Developer Environments ‚Äì Design & Implementation

Design Anchor
Containers define environments.
Editors are user choice.
Standardization is achieved by making the best path the easiest path.

Why This Exists

This repository exists to solve three recurring problems:

  • Developers lose time aligning local machines instead of writing code.
  • Rigid editor mandates create resistance and shadow workflows.
  • Powerful environments (Oracle, Java, Python, APIs) increase blast radius if poorly isolated.

The solution is not ‚Äúone tool for everyone,‚Äù but a disciplined separation of concerns.

Conceptual Model (Memorize This)

Layer 1 ‚Äì Workstation
Tools that help you work (Git, VS Code, Podman).
Layer 2 ‚Äì Containers
Reproducible, disposable development environments.
Layer 3 ‚Äì Editors
Windows into the same environment, nothing more.
Editors are intentionally outside the environment boundary so that:
  • UltraEdit users are not blocked
  • VS Code users gain extra convenience naturally
  • Secrets and credentials never live in editor state

Easy Mode: Developer Workstation Bootstrap

scripts/bootstrap-dev-workstation.ps1

This script exists to remove friction, not enforce policy. It installs recommended tools so the ‚Äúhappy path‚Äù is obvious, while leaving choice intact. 

# Installs:
# - Git
# - GitHub Desktop
# - VS Code
# - Podman Desktop
# - Notepad++ (Windows)
# - VS Code Remote / Containers / Java tooling
VS Code is recommended because it integrates deeply with containers.
It is not required.

macOS Helper: PowerShell Installation

scripts/install-powershell-macos.sh

PowerShell is the cross-platform automation layer. Bash installs PowerShell; PowerShell installs everything else. 

#!/usr/bin/env bash
set -e

if [[ "$OSTYPE" != "darwin"* ]]; then
  echo "macOS only"
  exit 1
fi

if ! command -v brew >/dev/null; then
  /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
fi

if ! command -v pwsh >/dev/null; then
  brew install --cask powershell
fi

pwsh -NoProfile -Command '$PSVersionTable.PSVersion'

OpenJDK 17 Dev Environment

Design Reasoning

Java 17 is LTS, container-aware, and stable. The container holds the JDK; the host does not.

openjdk17/Containerfile


FROM eclipse-temurin:17-jdk-jammy

RUN apt-get update && apt-get install -y \
    curl git bash openssh-client \
 && rm -rf /var/lib/apt/lists/*

WORKDIR /workspace
CMD ["bash"]

openjdk17/podman-compose.yml


services:
  openjdk17:
    image: openjdk17-dev
    volumes:
      - ./workspace:/workspace:Z
    tty: true
    stdin_open: true

PL/SQL + Python Dev Environment

Design Reasoning

Oracle and Python combine high power with high risk. Containers enforce structure; humans retain execution authority.

Dependency Strategy

Base dependencies are mandatory.
Optional dependencies are explicitly opt-in.

Optional Package Installation


OPTIONAL_PYTHON_PACKAGES=data,ml
This keeps the environment universal without forcing heavyweight libraries on every developer.

Security Rules (Non-Negotiable)

  • No secrets in Git
  • No secrets baked into images
  • No auto-execution against production systems
  • Containers prepare intent; humans execute intent
If you can explain it on paper,
you can defend it in a review,
and maintain it under pressure.

Final Memory Anchors

  • Easy mode installs tools
  • Containers define environments
  • Editors are replaceable
  • Optional means explicit intent

Ellucian Experience – Secure Podman Development Environment

This document describes the standardized, security‑focused local development environment for building Ellucian Experience cards using Podman Desktop and podman‑compose.

Design Goals
  • Support Ellucian Experience React/Node development
  • Run fully headless containers
  • Protect sensitive API credentials
  • Prevent privileged container execution
  • Work consistently on macOS, Windows, and Linux

Why Podman Desktop?

Podman uses a daemonless, rootless container model. Containers run as direct child processes of the invoking user and do not rely on a long‑running privileged daemon. This significantly reduces host risk compared to traditional daemon‑based container engines.

  • No root container execution
  • No privileged daemon
  • Per‑user container isolation
  • Strong fit for API‑driven development involving sensitive data

Podman Desktop is used strictly as a visual management client and does not weaken the underlying security posture. Containers remain headless and rootless.

Security Principles Applied

  • Rootless containers only
  • No secrets baked into images
  • Least privilege (capabilities dropped)
  • No privilege escalation
  • Headless container execution

This model aligns well with FERPA‑adjacent and ERP API integrations such as Ethos, where outbound, token‑based access is required without exposing services.

Project Layout

project-root/
├─ podman-compose.yml
├─ workspace/
│  ├─ Containerfile
│  └─ (Ellucian Experience extension source)
├─ env/
│  └─ .experience.env
├─ secrets/
│  └─ ethos-token.txt
└─ .gitignore

Important: .experience.env and secrets/ must never be committed to source control.

podman-compose.yml

version: "3.9"

services:
  experience-dev:
    image: ellucian/experience-dev:node18
    container_name: ellucian-experience-dev

    stdin_open: true
    tty: true

    user: "${UID:-1000}:${GID:-1000}"
    privileged: false

    cap_drop:
      - ALL

    security_opt:
      - no-new-privileges:true

    working_dir: /workspace

    volumes:
      - ./workspace:/workspace:Z
      - node-cache:/home/node/.npm
      - ./secrets:/run/secrets:ro,Z

    env_file:
      - ./env/.experience.env

    ports:
      - "3000:3000"

    command: ["bash"]

volumes:
  node-cache:

Containerfile

FROM node:18-bullseye

RUN apt-get update && apt-get install -y \
    git curl bash jq \
 && rm -rf /var/lib/apt/lists/*

RUN npm install -g \
    webpack \
    webpack-cli \
    typescript \
    eslint \
    bruno

WORKDIR /workspace

CMD ["bash"]

Running the Environment

podman build -t ellucian/experience-dev:node18 ./workspace
podman compose up

Developers may then bootstrap an Experience extension using: 

npx ellucian-create-experience-extension my-extension
cd my-extension
npm install
npm run deploy-dev

What This Environment Does NOT Do

  • No inbound public services
  • No privileged containers
  • No token storage in images
  • No Docker daemon

Intended Use

This environment is intended for:

  • Ellucian Experience card development
  • Ethos API integration testing
  • Secure frontend build pipelines

It is not intended as a production runtime or deployment target.

Maintained as a reference architecture for secure Ellucian Experience development using Podman Desktop.
